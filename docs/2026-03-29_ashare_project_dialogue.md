# 2026-03-29 A股项目讨论整理

## 文档目的

记录本次关于 `沪深主板 A 股 5 到 10 个交易日短周期研究系统` 的关键讨论结论，作为后续项目立项、设计和实现的统一口径。

## 已确认范围

- 市场范围：只做 `沪深主板`
- 研究周期：`5日` 与 `10日`
- 第一目标：做成普通用户可用的封装项目
- AI接入方式：直接接 `OpenAI API`
- 非目标：当前阶段不布局本地 AI，不做自动交易，不把美股逻辑直接套到 A 股

## 已确认的核心判断

### 1. 策略方向

第一版不做“单只股票价格点预测”，而做：

- `全市场短周期横截面选股排序`
- 每天收盘后对主板股票打分
- 输出未来 `5日` 与 `10日` 相对更强的候选股

选择这个方向的原因：

- A 股有 `T+1`、涨跌停、停牌、ST、新股前若干交易日特殊限制等规则
- `5到10天` 周期更适合做相对强弱排序，而不是绝对点位预测
- 面向普通用户做产品时，排序策略更稳定、可回测、可解释

### 2. 产品定位

目标不是做“神预测”，而是做一个：

- 可回测
- 可复现
- 可解释
- 可一键运行

的本地研究系统。

### 3. 系统边界

第一阶段先完成能研究、能产出、能验证的闭环：

- 行情采集
- 股票池过滤
- 市场扫描模块
- 选股模型
- 规则引擎
- 交易秘书
- 日志中心
- 最小验证器

第二阶段再补：

- 风控引擎
- 飞书同步器
- 可视化界面
- 复盘模块

第三阶段再做：

- 正式回测引擎
- 参数评估
- 模型管理
- 任务队列与缓存

### 4. AI 角色定位

AI 只能做增强层，不直接替代核心决策链：

- 市场摘要
- 候选股解释
- 日报生成
- 风险标签补充
- 复盘总结

核心决策链必须保持为：

`数据 -> 特征 -> 模型 -> 规则 -> 验证 -> 输出`

### 5. OpenClaw 判断

结论：`V1 不作为核心依赖`

原因：

- OpenClaw 更适合作为外层的 Agent 运行时与渠道集成层
- 适合接 Feishu、消息触发、任务分发、多 Agent 编排
- 不适合进入第一阶段的核心策略链路

推荐做法：

- 第一阶段只预留接口
- 第二阶段如需 Feishu 机器人或多 Agent 调度，再接 OpenClaw

### 6. TradingAgents 判断

结论：`适合借鉴，不适合直接当主引擎`

原因：

- 该框架默认偏 `yfinance / alpha_vantage`
- 主入口是 `单股票、单日期、多 Agent 讨论`
- 更像美股研究代理框架，不是 A 股全市场短周期选股引擎

推荐做法：

- 保留多 Agent 分工思想
- 只把它的思路用在“候选股解释层”
- 不让它直接生成最终买卖信号

## 本次讨论整理版

### 用户意图

用户希望做一个：

- 面向 A 股
- 预测周期为 `5到10个交易日`
- 面向普通用户可一键使用
- 引入 AI 协作能力

的封装项目。

### 讨论过程中的关键转折

1. 从“做 A 股预测”收敛到“做 A 股短周期选股排序系统”
2. 从“全 Agent 化”收敛到“核心链路确定性 + AI 增强层”
3. 从“本地 AI”收敛到“直接调用 OpenAI API”
4. 从“是否结合 OpenClaw / TradingAgents”收敛到“预留接入点，但不让其掌握主链路”

## 当前默认执行原则

- 只做 `沪深主板`
- 每日 `收盘后出信号，次日执行`
- 第一阶段必须包含最小验证闭环
- AI 不直接改分数、不绕过规则引擎
- 只接 `OpenAI API`
- 只把外部 Agent 框架作为可选解释层
